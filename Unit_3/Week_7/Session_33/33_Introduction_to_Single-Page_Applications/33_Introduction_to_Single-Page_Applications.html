Directives in AngularJS

AngularJS extends HTML with ng-directives.

The ng-app directive defines an AngularJS application.

The ng-model directive binds the value of HTML controls (input, select, textarea) to application data.

The ng-bind directive binds application data to the HTML view.

The ng-init directive initializes application data.

The ng-repeat directive repeats HTML elements for each item in a collection.

The ng-switch directive binds application data to groups of HTML elements.

The ng-style directive binds the values of CSS properties to application data.

The ng-if directive removes and recreates HTML elements based on application data.

The ng-show directive shows or hides HTML elements based on application data.

The ng-hide directive hides HTML elements based on application data.

The ng-click directive defines an AngularJS click event.

The ng-submit directive binds application data to the HTML submit event.

The ng-animate directive is used to animate HTML elements in AngularJS.

The ng-class directive is used to dynamically set CSS classes.

The ng-view directive is used to create Single Page Applications (SPA).

The ng-include directive is used to include HTML from an external file.

The ng-switch directive is used to conditionally swap DOM structure on your template based on a scope expression.

The ng-controller directive attaches a controller class to the view.

The ng-repeat directive repeats a set of HTML, a given number of times.

The ng-cloak directive is used to prevent the AngularJS html template from being briefly displayed by the browser in its raw (uncompiled) form while your application is loading.

The ng-bind-html directive binds application data to the HTML view, while sanitizing the input.

The ng-bind-template directive binds application data defined in the directive to the HTML view.

The ng-disabled directive binds application data to the disabled attribute of HTML elements.

The ng-open directive binds application data to the open attribute of the HTML details element.

The ng-readonly directive binds application data to the readonly attribute of HTML elements.

The ng-selected directive binds application data to the selected attribute of HTML elements.

The ng-src directive binds application data to the src attribute of the HTML img element.

The ng-srcset directive binds application data to the srcset attribute of the HTML img element.

The ng-href directive binds application data to the href attribute of the HTML a element.

The ng-trustAs directive is used to mark a HTML string as trusted.

Dependency Injecting in AngularJS

Dependency Injection is a software design pattern that deals with how components get hold of their dependencies.

The Angular injector subsystem is in charge of creating components, resolving their dependencies, and providing them to other components as requested.

AngularJS has built-in support for dependency injection which helps the developer to develop the application in a modular way and also helps in testing the application.

AngularJS provides the following ways to implement dependency injection:

Using the $injector object

Using the inline array annotation

Using the $inject property annotation

Using the $provide factory method

MVC in AngularJS

AngularJS is a MVC framework. MVC stands for Model View Controller.

Model: It is the lowest level of the pattern responsible for maintaining data.

View: It is responsible for displaying all or a portion of the data to the user.

Controller: It is a software Code that controls the interactions between the Model and View.

AngularJS uses the MVC pattern to separate the different aspects of the application (input logic, business logic, and UI logic), while providing a loose coupling between these elements.

AngularJS splits an application into MVC components in the following way:

How is Model implemenetd in AngularJS?

Model is implemented in AngularJS by using a JavaScript object. The JavaScript object contains the attributes that are used in the application.

How is View implemenetd in AngularJS?

View is implemented in AngularJS by using HTML. The HTML combines with the data and the result is displayed in the View.

How is Controller implemenetd in AngularJS?

Controller is implemented in AngularJS by using JavaScript functions. The JavaScript functions contain the business logic behind the application to which the scope objects refer to.

AngularJS provides the following features to support the Model View Controller pattern:

Scope: These are objects that refer to the model. They act as a glue between controller and view.

Controller: These are JavaScript functions that are bound to a particular scope.

Services: AngularJS comes with several built-in services for example $https: to make a XMLHttpRequests. These are singleton objects which are instantiated only once in app.

Filters: These select a subset of items from an array and returns a new array.

Directives: Directives are markers on DOM elements (such as elements, attributes, css, and more). These can be used to create custom HTML tags that serve as new, custom widgets. AngularJS has built-in directives (ngBind, ngModel...)

Templates: These are the rendered view with information from the controller and model. These can be a single file (like index.html) or multiple views in one page using "partials".

Routing: It is concept of switching views.
